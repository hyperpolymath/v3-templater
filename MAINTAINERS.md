## Becoming a Maintainer

### Path to Maintainer Status

1. **Start Contributing** (Perimeter 3)
   - Submit quality PRs
   - Help with issue triage
   - Review others' PRs
   - Improve documentation

2. **Become Regular Contributor** (~3 months)
   - Consistent high-quality contributions
   - Deep understanding of codebase
   - Helpful to community
   - 10+ merged PRs

3. **Invitation to Perimeter 2** (~6 months)
   - Demonstrated expertise
   - Trusted code review skills
   - Active community participation
   - Maintainer nomination

4. **Core Team Invitation** (~12 months)
   - Sustained contribution excellence
   - Leadership in project direction
   - Security mindset demonstrated
   - Unanimous maintainer approval

### Maintainer Responsibilities

- **Code Review**: Review PRs within 7 days
- **Issue Triage**: Respond to issues within 48 hours
- **Security**: Follow responsible disclosure
- **Community**: Enforce Code of Conduct
- **Documentation**: Keep docs current
- **Releases**: Participate in release process

## Decision-Making Process

### Consensus-Seeking

1. **Minor Changes**: Single maintainer approval
2. **Feature Additions**: Two maintainer approvals
3. **Breaking Changes**: All maintainers consensus
4. **Perimeter Changes**: Unanimous core team vote

### Conflict Resolution

1. Discussion in GitHub Issues/PRs
2. If no consensus: Lead Maintainer decision
3. For major disputes: Community RFC process
